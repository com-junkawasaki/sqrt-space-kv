Autoregressive decode stores a key-value (KV) cache growing linearly with sequence length S; on memory-constrained hardware this cache, not model weights, prevents a model from reaching the long context it was trained for. We study sqrt(S) residency: partition the sequence into blocks of stride b ~ sqrt(S log S), keep only boundary checkpoints plus one active block resident, and page the rest from host memory or disk, combined with online softmax so attention stays exact. The checkpoint-and-recompute pattern is classical -- the reverse-mode-differentiation checkpointing of Griewank-Walther and Chen et al.; we borrow Williams' (STOC 2025) improved stride formula as a principled default block size, not a claim that Transformer decode is a multitape Turing machine. We validate the full chain: (1) correctness end-to-end, via a real Cache implementation fused into a real generation loop, byte-identical to full-KV decode; (2) honest cost -- re-paging every step is 2.2x-3.7x slower, measured two independent ways on NVIDIA A100, worsening with S; (3) at the ~90% compression this method's own headline claims, three lossy eviction baselines (StreamingLLM, a norm scorer, SnapKV) score 0% accuracy on needle retrieval where sqrt(S) residency is exact by construction; (4) on Apple Silicon unified memory, naive host offloading does not free usable memory, while memory-mapped paging does, confirmed to 6.3 GB at 0.0 MB RSS cost; and (5) a real 9B model with a 1.05M token native context reaches only 6.1% of that context on a 16 GB Mac without this technique. Negative results, reported transparently: whole-prefix recompute is not a peak-memory win, and neither the HuggingFace nor MLX reference implementation of DeepSeek-V2's Multi-head Latent Attention caches its compressed latent, so composability with an absorbed-cache MLA remains open.
