Williams (STOC 2025) showed that time-t multitape Turing machines can be simulated in O(sqrt(t log t)) space via Tree Evaluation and Cook-Mertz. We transfer the same block-size optimization to LLM serving: treat decode length S as time, set KV checkpoint stride b ≈ sqrt(S log S), and retain only boundary checkpoints plus one active block (√S residency). We do not claim Transformers are multitape TMs. Empirically, on Apple Silicon (SmolLM-135M, Qwen2.5-0.5B, gemma4-E4B) and Modal NVIDIA A100 (Qwen2.5-7B/14B bf16), full past_key_values grow linearly in S at a constant bytes-per-token fixed by architecture. √S residency reduces KV storage by 90-97% for S in [1k, 16k]; at S=16k, 7B full KV is 940 MB vs 25 MB resident, and 14B is 3.22 GB vs 87 MB (~3.1 GB saved on the KV budget). Host-paged online-softmax attention over blocks remains numerically exact. Naive whole-prefix recompute each token increases peak device memory because activations dominate. Process-level RSS can mask KV savings when weights dominate. We release a co-scientist tournament of memory policies, pure cost models, Mac and Modal runners, and benchmarks in gftdcojp/cloud-murakumo.
